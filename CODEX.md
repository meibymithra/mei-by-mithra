# CODEX.md

## Working assumptions for this repo

- Treat the website as a brand-led sales and trust surface for Mithra Krishnamoorthy
- The admin area exists for operations, but the public experience must feel premium and customer-first
- The store, booking flow, intake flow, and testimonial flow are all part of one coherent public system

## Design stance

- Follow the existing brand guideline before introducing new UI motifs
- Use motion carefully: atmosphere, reveal, and emphasis are good; novelty without clarity is not
- Favor elegant editorial composition over template-driven marketing blocks
- Make the UI memorable through rhythm, surfaces, linework, layering, and brand tone rather than random effects

## Product stance

- A visitor should quickly understand:
  - who Mithra is
  - what kinds of work she does
  - how to book
  - how the intake process works
  - how to buy a playbook
- Feedback and testimonials must remain moderation-controlled
- Package or multi-session intent should be supported in the workflow

## Documentation stance

- If you change any public flow, update `docs/SYSTEM.md`
- If you change visual or messaging direction, update `docs/BRAND.md`
- If you change deployment or integration behavior, update `docs/DEPLOYMENT.md` and related docs
